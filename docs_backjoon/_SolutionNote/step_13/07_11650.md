### 메모

- `std::sort`는 비교함수(Comparator)만 있으면 정렬이 가능하며 `std::pair`는 표준 라이브러리에서 **`operator<`가 이미 정의**되어 있어서, 별도 비교 함수를 안 줘도 정렬이 됨

- 비교 함수가 없거나 다른 비교함수를 사용하고 싶을 경우 `std::sort`의 세번째 인수에 비교함수를 넣으면 됨

#### 정렬 기준 변경

```cpp
std::sort(a.begin(), a.end(),
    [](const auto& lhs, const auto& rhs)
    {
        if (lhs.second != rhs.second)
        {
            return lhs.second < rhs.second;
        }
        
        return lhs.first < rhs.first;
    });
```

- `comp(lhs, rhs) == true` → `lhs`가 `rhs`보다 앞에 와야 함
    
- `false` → 그 반대(또는 동등)

### GPT 피드백

- `std::sort(A.begin(), A.end())`만으로 11650 요구사항( x 오름차순, x 같으면 y 오름차순 )이 충족됨 (`pair`의 `operator<`가 lexicographical 비교)
    
- `for (size_t i = 0; i < N; ++i)`는 `N`이 `int`라서 signed/unsigned 혼용 경고 가능성이 있음. `int i`로 맞추거나 `N`을 `size_t`로 받는 편이 일관적임
    
- `vector`에 `reserve(N)`를 넣으면 입력이 큰 경우 재할당을 줄일 수 있음(필수는 아님)
    
- 출력 루프에서 `for (const auto& a : A)`로 참조 순회하면 불필요한 복사를 피할 수 있음
  (정답엔 영향 없음)

### 정석 코드 제안
#### 정석 코드
```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <utility>

int main()
{
    std::ios::sync_with_stdio(false);
    std::cin.tie(nullptr);

    int n;
    std::cin >> n;

    std::vector<std::pair<int, int>> a;
    a.reserve(n);

    for (int i = 0; i < n; ++i)
    {
        int x, y;
        std::cin >> x >> y;
        a.push_back({ x, y });
    }

    std::sort(a.begin(), a.end());

    for (const auto& p : a)
    {
        std::cout << p.first << ' ' << p.second << '\n';
    }

    return 0;
}
```
