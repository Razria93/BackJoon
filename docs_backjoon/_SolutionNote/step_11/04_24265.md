### GPT 피드백

- 첫 줄 `n(n-1)/2`는 코드1 실행 횟수 `f(n)`에 해당하며, 둘째 줄 `2`는 최고차항 차수로 정확함
    
- `size_t`는 무부호 타입이라 `N-1`에서 N=0 같은 입력이 오면 언더플로우가 발생하지만, 문제 입력은 보통 `n ≥ 1`로 주어지므로 실질 문제는 없음
    
- 안전하게 쓰려면 `long long`을 사용해 오버플로우 리스크를 줄이는 편이 일반적임(특히 `N*(N-1)`)

---
### 정석 코드 제안
#### 정석 코드
```cpp
#include <iostream>

int main()
{
    long long n;
    std::cin >> n;

    std::cout << (n * (n - 1)) / 2 << '\n' << 2;
    return 0;
}
```
