### 메모

- - `"factor"` / `"multiple"`의 기준은 **항상 A**
    
- `0 0`은 **데이터가 아니라 제어 신호**이므로 연산 전에 반드시 종료 판정
    
- `% (나머지)` 연산은 **분모가 0이면 UB** → 종료 판정 이후에만 연산해야 안전

| 유형                 | 예시                                       |
| ------------------ | ---------------------------------------- |
| `0 0` 종료 (제어신호 제공) | `if(A == 0 && B == 0) break;`            |
| EOF 종료             | `while(cin >> x)`                        |
| N회 반복              | 첫 줄에서 N 체크 / `for문` 사용                   |
| sentinel 문자열       | `END`, `QUIT`의 문자열과 입력된 문자열을 비교 후 같으면 종료 |

### GPT 피드백

- 문제의 본질은 문자열 출력이 아니라 **정수의 나눗셈 관계 분류**
    
- 입력 루프에서 **입력 → 종료 판정 → 연산** 순서를 지키지 않으면 런타임 오류 가능
    
- 조건문 순서는 의미적으로 `B % A == 0`(factor) → `A % B == 0`(multiple)이 자연스럽고 안전

### 정석 코드 제안
#### 정석 코드
```cpp
#include <iostream>

int main()
{
    std::ios::sync_with_stdio(0);
    std::cin.tie(0);

    size_t A, B;

    while (true)
    {
        std::cin >> A >> B;

        if (A == 0 && B == 0)
            break;

        if (B % A == 0)
        {
            std::cout << "factor\n";
        }
        else if (A % B == 0)
        {
            std::cout << "multiple\n";
        }
        else
        {
            std::cout << "neither\n";
        }
    }

    return 0;
}
```
