### 메모

- **더 적은 개수의 봉지** 를 구하는 문제이므로 **5kg** 을 우선적으로 소모한 뒤 **3kg** 을 소모하는 아이디어를 가지고 접근

- 하지만 더 나은 아이디어가 있으므로 해당 방식으로 재풀이 요망

	- 해당 아이디어는 `n > 0` 일 때 반복문으로 실행함
	
	- `n % 5 == 0` 조건을 우선적으로 체크하며 해당 조건을 만족할 때, 즉 `n` 이 5의 배수 일 때 5를 한번에 처리함
	
	- `n` 이 5의 배수가 되기 전까지는 `n -= 3` 으로 `n`값을 줄여나감

### GPT 피드백

- 아이디어는 정답이 맞고, `N/5`까지만 루프를 도는 것도 상한이 명확해서 안전함
    
- 다만 현재 방식은 “5를 몇 개 쓰는지”를 전부 시도하므로, 이 문제의 정석인 **그리디(5부터 최대한 쓰고 3으로 조정)**보다 구현이 조금 무겁지만 여전히 충분히 빠름
    
- `min`이라는 변수명은 `<algorithm>`의 `std::min`과 이름이 겹쳐 혼동을 만들 수 있음
  (컴파일은 되더라도 가독성/충돌 위험) → `minCount` 같은 이름 권장
    
- `std::min`을 쓰려면 원칙적으로 `#include <algorithm>`이 필요함
    
- `result` 플래그 없이도 `minCount`가 갱신됐는지로 판정 가능함(구조 단순화 가능)

### 정석 코드 제안
#### 정석 코드
```cpp
#include <iostream>

int main()
{
    std::ios::sync_with_stdio(false);
    std::cin.tie(nullptr);

    int n;
    std::cin >> n;

    int count = 0;

    while (n > 0)
    {
        if (n % 5 == 0)
        {
            count += n / 5;
            n = 0;
            break;
        }

        n -= 3;
        ++count;
    }

    if (n == 0)
    {
        std::cout << count;
    }
    else
    {
        std::cout << -1;
    }

    return 0;
}
```
