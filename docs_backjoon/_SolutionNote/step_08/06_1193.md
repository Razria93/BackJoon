### 메모

- 각 대각선은 원소 개수가 1, 2, 3, … 순으로 증가함
    
- N번째 값은 `k(k+1)/2 ≥ N` 을 만족하는 최소의 `k`번째 대각선에 포함됨
    
- 대각선 번호의 **홀짝 여부**에 따라 분수가 증가하는 방향이 반전됨
    
- 좌표나 벡터 개념 없이 **순수 인덱스 산술**로 해결 가능한 문제
  
---
### GPT 피드백

- 이 문제는 분수의 실제 위치나 좌표를 계산하는 문제가 아니라, **대각선 레벨 판별 문제**로 환원하는 것이 핵심임
    
- 누적합을 이용해 대각선을 찾고, 내부 인덱스를 계산하는 방식이 가장 단순하고 안정적임
    
- 대각선 번호(`count + 1`)의 홀짝으로 출력 순서를 분기하는 구조는 문제 의도와 정확히 일치함

### 정석 코드 제안
#### 정석 코드
```cpp
#include <iostream>

int main()
{
    std::ios::sync_with_stdio(0);
    std::cin.tie(0);

    size_t N;
    std::cin >> N;

    size_t count = 1;

    // 1) N이 포함된 대각선 찾기
    while (true)
    {
        size_t sum = count * (count + 1) / 2;

        if (N <= sum)
            break;

        ++count;
    }

    // 2) 해당 대각선 내부 인덱스 계산
    size_t container_idx = count + 1;
    size_t prev_sum = (count - 1) * count / 2;
    size_t idx = N - prev_sum;
    size_t value = container_idx - idx;

    // 3) 대각선 번호 홀짝에 따라 출력 방향 결정
    if ((container_idx % 2) == 1)
        std::cout << idx << '/' << value;
    else
        std::cout << value << '/' << idx;

    return 0;
}
```
